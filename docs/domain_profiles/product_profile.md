# Product Profile

Use this profile for product, feature, and specification documents.

## Optional fields

- `product_family`
- `benefit`
- `rider`
- `guarantee`
- `charge`
- `eligibility`
- `premium_rule`
- `policyholder_option`
- `system_requirement`
- `valuation_impact`
- `pricing_impact`

## Guidance

- Keep feature and eligibility details separate from valuation or pricing
  consequences.
- Preserve the product context that drives model or filing decisions.
- Use this profile for product specifications, feature inventories, and
  product-support evidence.
