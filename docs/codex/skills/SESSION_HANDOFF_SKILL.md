# Session Handoff Skill

## Purpose

Preserve context across Codex sessions so the processor continues from the
current state instead of restarting.

## Rules

- Start by reading `docs/project-state/CURRENT_STATE.md`.
- Read `docs/project-state/HANDOFF_PROCESS.md`,
  `docs/project-state/SOURCE_GUARDRAILS.md`, `docs/project-state/NEXT_ACTIONS.md`,
  `docs/project-state/DECISIONS.md`, and `docs/project-state/SESSION_LOG.md`.
- Run `git status --short --branch` before editing.
- Inspect the relevant config, scripts, and docs before making changes.
- Update the project-state docs before ending the session.
- Preserve the raw-source separation and source-bound guardrails at all times.
