# Privacy and Data Hygiene Skill

## Purpose

Protect sensitive material and keep Git clean.

## Rules

- Do not commit raw internal documents, proprietary pricing files, liability
  model documentation, videos, large PDFs, or sensitive data.
- Use manifests, metadata, sanitized summaries, and lightweight exports
  instead.
- Flag any file that appears sensitive before processing or exporting.
- Keep raw material in the external source folder rather than in this repo.
