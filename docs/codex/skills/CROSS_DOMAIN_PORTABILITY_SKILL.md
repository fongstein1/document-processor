# Cross-Domain Portability Skill

## Purpose

Keep the processor reusable across multiple actuarial and regulatory domains.

## Rules

- Support NAIC regulation documents, pricing documents, liability modeling
  documents, and future governance/process documents.
- Keep domain-specific assumptions isolated in config files.
- Avoid hard-coding NAIC-only logic into the processor core.
- Prefer reusable metadata fields over one-off source-family branches.
